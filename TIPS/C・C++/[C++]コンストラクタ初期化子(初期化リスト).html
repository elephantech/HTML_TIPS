コンストラクタ初期化子(初期化リスト)とは

コンストラクタの実処理に先立ってメンバ変数を初期化する際に用いる、初期化対象と値の対。

初期化リストを用いる理由

以下の各節で挙げる理由により、メンバ変数の初期化は初期化リストを用いることが望ましい。

処理が高速となる


上記のクラス定義におけるインスタンス生成結果は一見変わらない。
しかし、初期化リストを用いない場合、上記の m_a = x; 文は、
  1. 変数 m_a を int 型のデフォルト値(不定)で「初期化」する
  2. 変数 m_a に x を 「代入」する
という二つのステップを経過するが、初期化リストを用いる場合は、
  1. 変数 m_a を xで 「初期化」する
となり、初期化リストを用いたほうが踏む手順が減る。
特に、メンバ変数がクラス(または構造体)インスタンスで、コンストラクタの処理が重い場合に威力を発揮する。
また、次節以降で述べるが、コンストラクタの実装によっては初期化リストを用いる以外での初期化自体が不可能となる。

メンバ変数がクラスの場合

// 引数付きコンストラクタをもつCHogeクラス class CHoge { public: // (引数付き)コンストラクタ CHoge(int x) { (略) } } // CHogeクラスをメンバにもつCFugaクラス class CFuga { private: CHoge m_cl_Hoge; //< CHogeクラスのインスタンス public: CFuga(int x) { m_cl_Hoge(x); //< CHogeクラスのインスタンスを初期化しようとしているが… } }

上記のコードはエラーとなる 。
前節で説明した通り、CFuga クラスのインスタンス生成時にはメンバ変数 m_cl_Hoge が 「デフォルトコンストラクタ(引数なしのコンストラクタ)」で初期化される が、
CHoge クラスには「引数なしのコンストラクタ」が存在しない ためである。
このように、 メンバ変数がクラス(または構造体)インスタンスかつ、デフォルトコンストラクタが存在しない場合は、初期化リストを用いる以外に初期化の手段は無い 。

以下、修正したコード。

// 引数付きコンストラクタをもつHogeクラス class CHoge { public: // (引数付き)コンストラクタ CHoge(int x) { (略) } } // HogeクラスをメンバにもつFugaクラス class CFuga { private: CHoge m_cl_Hoge; //< Hogeクラスのインスタンス public: CFuga(int x) : m_cl_Hoge(x) //< 引数付きコンストラクタで初期化される { (略) } }

constメンバ変数の初期化

const修飾されたメンバ変数は、初期化リストを用いなければ初期化できない。(代入が許可されないため。)
constメンバ変数については、[C／C++]const修飾の使い方 を参照のこと。

「変数の初期化」と「生成時の付随処理」を分離できる

コンストラクタ内にメンバ変数の初期化と、それ以外の処理が混在する場合、クラスが大きくなるにつれてそれらの峻別は難しくなる。
初期化リストを用いることで両者を分離できるため、コードの可読性向上にもつながる。
