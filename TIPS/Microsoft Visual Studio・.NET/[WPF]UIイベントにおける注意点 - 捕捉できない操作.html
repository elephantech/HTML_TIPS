WPFにおけるユーザ操作の受付

WPFアプリケーションにおいてマウスやキーボードなどの操作を捕捉しようとしても、対象となるコントロール( = イベントセンダ)や押下したキーによっては
本来発生するはずのイベント( MouseDown 、KeyDown など)が発生しない。
その場合は、対となる Preview イベントを捕捉することで代用が可能である。

Preview イベントとは

ユーザ操作によって発生するイベントのうち、本イベントに先だって発生するもの。
例えばマウスボタンを押下した場合、本イベントの MouseDown イベントの前に PreviewMouseDown イベントが発生する。
これらはルーティングイベントとして連携されており、Preview イベントにてイベント引数を「処理済み」
( EventArgs.Handled = true )
とすることで、後続の本イベントの発生を抑制することができる。

Preview イベントの例

Preview イベントの一部を下表に記す。
ユーザ操作 Preview イベント 対となる本イベント
キー押下 PreviewKeyDown KeyDown
キー戻り PreviewKeyUp KeyUp
キーボードフォーカス取得 PreviewGotKeyboardFocus GotKeyboardFocus
キーボードフォーカス喪失 PreviewLostKeyboardFocus LostKeyboardFocus
マウスダブルクリック PreviewMouseDoubleClick MouseDoubleClick
マウスボタン押下 PreviewMouseDown MouseDown
マウスボタン戻り PreviewMouseUp MouseUp
マウス左ボタン押下 PreviewMouseLeftButtonDown MouseLeftButtonDown
マウス左ボタン戻り PreviewMouseLeftButtonUp MouseLeftButtonUp
マウス右ボタン押下 PreviewMouseRightButtonDown MouseRightButtonDown
マウス右ボタン戻り PreviewMouseRightButtonUp MouseRightButtonUp
マウスホイール操作 PreviewMouseWheel MouseWheel
マウス移動 PreviewMouseMove MouseMove
文字入力 PreviewTextInput TextInput
