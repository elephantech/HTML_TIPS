C#のイベント機能

モジュール間、またはタスク間の通信にイベントを用いることができる。

例1 : データを返さないイベント

SleepClass.cs
using System; // 実験用クラス public class SleepClass { // イベントデリゲート Time の宣言 // (delegate ではなく event を用いる点に注意) public event EventHandler Time; // 実験用メインルーチン public void Start() { System.Threading.Thread.Sleep(5000); if (Time != null) { // Time イベントの発生 // イベントセンダは自身、イベント変数は空(Empty) Time(this, EventArgs.Empty); } } }
HogeWindow.cs
using System; // button1 押下時の処理 private void button1_Click(object sender, EventArgs e) { SleepClass clsSleep = new SleepClass(); // Time イベント発生時のハンドラとして // Callback() を登録 clsSleep.Time += new EventHandler(this.Callback); // 実験用メインルーチン開始 clsSleep.Start(); } // イベントハンドラとして登録するコールバック private void Callback(object sender, EventArgs e) { // イベントが発生したとき MessageBox.Show("OK!"); }

何が起きるか

  1. ユーザが HogeWindow の button1 を押下する
  2. (HogeWindow) button1_Click() が実行され、SleepClass.Time イベントのハンドラとして Callback() が登録される
  3. (SleepClass) Start() が実行される
  4. Start() 内で Time イベントが発行される
  5. イベントハンドラとして登録された Callback() が実行される
  6. 結果として、メッセージボックスが表示される

例2 : データを返すイベント

イベントレシーバにデータを返す場合を考える(例1の改良)。
SleepClass.cs
using System; // Time イベントで返されるデータ // System.EventArgs を継承 public class TimeEventArgs : EventArgs { public string Message; } // 実験用クラス public class SleepClass { // イベントデリゲート Time の宣言 // (delegate ではなく event を用いる点に注意) public event EventHandler Time; // 実験用メインルーチン public void Start() { System.Threading.Thread.Sleep(5000); if (Time != null) { // Time イベントの発生 // イベントセンダは自身、イベント変数を設定 TimeEventArgs e = new TimeEventArgs(); e.Message = "終わったよ。"; Time(this, e); } } }
HogeWindow.cs
using System; // button1 押下時の処理 private void button1_Click(object sender, EventArgs e) { SleepClass clsSleep = new SleepClass(); // Time イベント発生時のハンドラとして // Callback() を登録 clsSleep.Time += new EventHandler(this.Callback); // 実験用メインルーチン開始 clsSleep.Start(); } // イベントハンドラとして登録するコールバック private void Callback(object sender, EventArgs e) { // イベントが発生したとき MessageBox.Show(e.Message); }
例1との違いは、イベント変数として用いる TimeEventArgs クラスを追加したことと、
TimeEventArgs 型のオブジェクトを用いて、イベントセンダからイベントレシーバへデータを渡していることである。

あとがき

イベントを用いることで、関連するモジュール同士の関係を疎結合に保つことができ、コードの見通しが良くなる。
特にUIとミドルとの連携時にもミドルに対してUIのハンドラを渡す必要が無く、レイヤの境界が曖昧とならない。
また、ひとつのイベントに対してイベントハンドラ(コールバック)は複数登録することができ、発生時には登録された順に実行される。
(加算代入演算子を用いるのも「追加登録」を行うため。)
